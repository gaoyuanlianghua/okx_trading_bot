# OKX 交易机器人

## 项目介绍

OKX 交易机器人是一个基于事件驱动的智能体系统，支持 OKX 交易所的 REST API 和 WebSocket 连接。该项目采用新的架构设计，包含市场数据、订单管理、风险管理和策略执行等核心模块，专注于交易和收益跟踪功能。

### 主要功能

- OKX REST API 和 WebSocket 客户端支持
- WebSocket 连接管理和自动重连
- 事件驱动的智能体系统
- 交易和收益跟踪功能
- 账户信息和资产分布管理
- 风险管理和风险警报
- 策略集成（包括原子核互反动力学策略）
- 交易摘要功能
- 健康检查机制
- 指数退避重试机制
- 优化的线程管理
- 实时监控和日志系统
- 详细的 API 调用和响应监控
- 策略运行监控和性能跟踪
- WebSocket GUI 界面

## 安装与运行

### 系统要求

- Windows 10/11
- Python 3.8+
- 至少 2GB RAM
- 至少 100MB 可用磁盘空间

### 依赖安装

```bash
pip install -r requirements.txt
```

### 直接运行

#### 命令行模式

```bash
python main_new.py
```

#### GUI模式

```bash
python websocket_gui.py
```

### 打包运行

#### 打包命令

```bash
pyinstaller --onefile --windowed --name okx_trading_bot websocket_gui.py
```

#### 运行打包后的程序

直接双击生成的 `okx_trading_bot.exe` 文件即可运行。

- **命令行模式**：通过环境变量或配置文件设置 API 信息

## 使用说明

### 启动机器人

#### GUI模式
1. 运行 `python websocket_gui.py` 启动 GUI 界面
2. 在顶部输入框中输入您的 OKX API 密钥、密钥密码和密码短语
3. 选择模拟盘或实盘环境
4. 点击 "连接" 按钮连接到 OKX 服务器
5. 连接成功后，系统会自动订阅默认产品（BTC-USDT-SWAP、ETH-USDT-SWAP）

#### 命令行模式
1. 运行 `python main_new.py` 启动命令行模式
2. 系统会自动初始化所有组件并启动交易机器人

### 策略管理

1. 在 GUI 界面的 "策略管理" 标签页中，可以查看、创建、启动和停止策略
2. 点击 "创建策略" 按钮，输入策略名称和类型，然后粘贴策略代码
3. 选择策略后，点击 "启动策略" 按钮启动策略
4. 策略运行状态会实时显示在策略列表中

### 监控机器人

- **市场数据**标签页：显示实时行情数据
- **订单**标签页：显示订单状态
- **账户**标签页：显示账户信息和资产分布
- **订阅管理**标签页：管理 WebSocket 订阅
- **监控**标签页：显示策略监控、API 监控和日志信息

### 停止机器人

- **GUI模式**：直接关闭程序窗口
- **命令行模式**：按 Ctrl+C 停止程序

## 帮助与支持

### 常见问题

1. **连接失败**
   - 检查 API 密钥是否正确
   - 检查网络连接
   - 检查防火墙设置

2. **GUI 冻结**
   - 等待一段时间，机器人会自动恢复
   - 或尝试重启程序

3. **WebSocket 断开**
   - 机器人会自动重连
   - 检查网络稳定性

4. **如何添加自定义策略**
   - 在 "策略管理" 标签页点击 "创建策略" 按钮
   - 输入策略名称和类型
   - 在弹出的对话框中粘贴策略代码
   - 点击 "创建" 按钮

### 日志查看

日志文件位于 `logs/` 目录下，按日期和大小轮换。

### 联系方式

如有问题或建议，请通过 GitHub Issues 提交。

### 架构说明

#### 1. 应用层
- **main_new.py**: 命令行模式入口，负责初始化和启动整个交易机器人
- **websocket_gui.py**: GUI 模式入口，提供可视化交易操作和监控界面

#### 2. 智能体层（核心业务逻辑）
- **协调智能体**: 中央协调器，管理其他智能体的协作和资源分配
- **市场数据智能体**: 负责获取和处理市场数据，订阅行情信息
- **订单智能体**: 处理订单的创建、更新、取消等操作
- **策略智能体**: 执行交易策略，生成交易信号
- **风险管理智能体**: 监控风险指标，实施风险控制措施

#### 3. 监控层
- **策略监控器**: 监控策略运行状态，收集性能指标，记录交易历史

#### 4. 策略层
- **策略基类**: 定义策略接口和通用方法
- **动态策略**: 实现动态交易策略逻辑
- **Passivbot集成**: 集成Passivbot自动交易系统

#### 5. 基础设施层
- **事件总线**: 实现智能体间的消息通信机制
- **智能体通信协议**: 定义智能体间的通信规范

#### 6. API层
- **OKX REST客户端**: 封装OKX REST API调用，支持批量操作和详细的API监控
- **WebSocket客户端**: 处理WebSocket连接和实时数据，支持自动重连和消息监控

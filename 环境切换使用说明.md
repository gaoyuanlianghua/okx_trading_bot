# 实盘和模拟盘分离使用说明

## 概述

系统现在已实现实盘和模拟盘的完全分离，您可以安全地在模拟盘中测试策略，而不会影响实盘资金。

## 配置文件说明

### 配置文件位置

所有配置文件位于 `config/` 目录下：

- `config_live.yaml` - 实盘配置文件
- `config_test.yaml` - 模拟盘配置文件
- `current_env.json` - 当前环境状态文件（自动生成）
- `config.yaml` - 旧配置文件（保留用于向后兼容）

### 配置模拟盘

1. 登录 OKX 官网
2. 进入 "交易" → "模拟交易"
3. 点击 "个人中心" → "创建模拟盘APIKey"
4. 创建 API 密钥后，编辑 `config/config_test.yaml`：

```yaml
api: 
  api_key: "您的模拟盘API Key"
  api_secret: "您的模拟盘API Secret"
  is_test: true
  passphrase: "您的模拟盘Passphrase"
  timeout: 30
```

**重要提示**：
- 模拟盘的 API 密钥与实盘是完全独立的
- 模拟盘使用独立的资金账户，不会影响实盘
- 模拟盘部分功能不支持（如提币、充值等）

## 环境切换脚本

### 使用方法

系统提供了 `switch_env.py` 脚本来方便地切换环境：

#### 1. 查看当前环境信息


#### 2. 切换到实盘

**注意**：切换到模拟盘前，请确保已配置好 `config/config_test.yaml` 中的模拟盘 API 密钥。

## 程序内部使用

### 环境管理器 API

如果您在代码中需要使用环境信息，可以使用 `env_manager`：

```python
from core.config.env_manager import env_manager

# 检查当前环境
if env_manager.is_live_env():
    print("当前在实盘环境")
elif env_manager.is_test_env():
    print("当前在模拟盘环境")

# 获取当前环境类型
current_env = env_manager.get_current_env()

# 获取API配置
api_config = env_manager.get_api_config()
print(f"API Key: {api_config['api_key']}")
print(f"是否模拟盘: {api_config['is_test']}")

# 获取环境信息
env_info = env_manager.get_env_info()
```

### API 客户端自动适配

系统的 API 客户端（`OKXRESTClient` 和 `OKXWebSocketClient`）已经自动支持实盘和模拟盘：

- **REST API**：自动添加 `x-simulated-trading: 1` 头（模拟盘时）
- **WebSocket**：自动使用正确的 WebSocket 地址
  - 实盘：`wss://ws.okx.com:8443`
  - 模拟盘：`wss://wspap.okx.com:8443`

## 注意事项

1. **API 密钥安全**：
   - 不要将包含真实 API 密钥的配置文件提交到版本控制系统
   - `config_test.yaml` 已设置占位符，需要您手动填入真实的模拟盘密钥

2. **环境切换后重启**：
   - 切换环境后，建议重启交易程序以确保使用新配置

3. **测试策略**：
   - 在实盘交易前，务必在模拟盘中充分测试策略
   - 模拟盘和实盘的市场数据是一样的，但订单执行可能有细微差异

4. **资金隔离**：
   - 模拟盘和实盘使用完全独立的资金账户
   - 模拟盘交易不会产生真实的资金损失或收益

## 常见问题

**Q: 如何确认当前在哪个环境？**

A: 运行 `python3 switch_env.py info` 查看当前环境信息。

**Q: 模拟盘 API 密钥在哪里创建？**

A: 在 OKX 官网的 "交易" → "模拟交易" → "个人中心" → "创建模拟盘APIKey"。

**Q: 切换环境后需要做什么？**

A: 建议重启交易程序，以确保使用新的配置。

**Q: 模拟盘和实盘的区别是什么？**

A: 
- 模拟盘使用虚拟资金，不产生真实盈亏
- 模拟盘 API 使用独立的密钥
- 模拟盘部分功能不可用（如提币、充值）
- 模拟盘 WebSocket 使用不同的地址

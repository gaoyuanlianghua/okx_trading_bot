## 3. 交易执行完整流程

### 3.1 流程概览

```
1. 市场数据获取
   └── MarketDataAgent/WebSocket ─► StrategyAgent._execute_strategy_cycle()

2. 策略执行
   └── StrategyAgent.execute(market_data) ─► BaseStrategy.execute()
   └── BaseStrategy._execute_strategy() ─► NuclearDynamicsStrategy._execute_strategy()

3. 信号生成
   └── NuclearDynamicsStrategy._generate_signal() ─► 返回信号字典

4. 信号处理
   └── StrategyAgent._process_signal() ─► 发布STRATEGY_SIGNAL事件

5. 交易决策
   └── CoordinatorAgent._on_strategy_signal() ─► 构建订单参数

6. 订单执行
   └── CoordinatorAgent.place_order() ─► OrderAgent.place_order()

7. 交易所执行
   └── OrderAgent ─► OKX REST API
```

## 4. 关键类和方法

### 4.1 StrategyAgent (策略智能体)

**文件**: `core/agents/strategy_agent.py`

| 方法 | 功能 | 调用关系 |
|------|------|---------|
| `execute()` | 执行策略周期 | 被定时器调用 |
| `_execute_strategy_cycle()` | 获取市场数据并执行策略 | 调用`strategy.execute()` |
| `_process_signal()` | 处理策略信号并发布事件 | 发布`STRATEGY_SIGNAL`事件 |
| `_process_signal_batch()` | 批处理信号 | 定时调用`_process_single_signal` |
| `_process_single_signal()` | 处理单个信号 | 发布`RISK_ALERT`和`STRATEGY_SIGNAL`事件 |

**关键调用链**:
```python
execute() 
  → _execute_strategy_cycle(strategy_name)
    → strategy.execute(market_data)  # 调用策略
      → _process_signal(signal)
        → EventBus.publish(STRATEGY_SIGNAL)
```

### 4.2 BaseStrategy (策略基类)

**文件**: `strategies/base_strategy.py`

| 方法 | 功能 |
|------|------|
| `execute(market_data)` | 执行策略入口，记录执行日志 |
| `_execute_strategy(market_data)` | 子类必须实现的策略执行逻辑 |
| `_log_execution()` | 记录策略执行日志 |

### 4.3 NuclearDynamicsStrategy (核动力策略)

**文件**: `strategies/nuclear_dynamics_strategy.py`

| 方法 | 功能 |
|------|------|
| `_execute_strategy(market_data)` | 执行策略主逻辑，**必须返回signal** |
| `_generate_signal()` | 生成交易信号 |
| `_calculate_spring_drift()` | 计算弹簧飘移 |
| `_calculate_angular_momentum_flow()` | 计算角动量流 |
| `_calculate_pairing_gap()` | 计算配对能隙 |
| `_calculate_phase_sync()` | 计算相位同步 |

**信号结构**:
```python
{
    "strategy": "NuclearDynamicsStrategy",
    "side": "buy" | "sell" | "neutral",
    "signal_level": "S" | "SS" | "SSS",
    "signal_strength": 1.0,
    "signal_score": 100,
    "price": float,
    "timestamp": float,
    "expected_return": float,
    "inst_id": "BTC-USDT"
}
```

### 4.4 CoordinatorAgent (协调智能体)

**文件**: `core/agents/coordinator_agent.py`

| 方法 | 功能 |
|------|------|
| `_on_strategy_signal()` | 接收并处理策略信号，构建订单参数 |
| `_get_available_usdt_balance()` | 获取USDT余额 |
| `_get_account_info()` | 获取账户信息 |
| `place_order()` | 调用订单智能体执行下单 |

**关键调用链**:
```python
_on_strategy_signal(event)
  → 检查信号类型（buy/sell/neutral）
  → 计算交易数量和价格
  → 构建order_params
  → order_agent.place_order(order_params)
```

### 4.5 OrderAgent (订单智能体)

**文件**: `core/agents/order_agent.py`

| 方法 | 功能 |
|------|------|
| `place_order(params)` | 执行下单，包含收益率检查 |
| `_execute_market_order()` | 执行市价单 |
| `_handle_trade_result()` | 处理交易结果 |

**skip_return_check参数**: 用于跳过收益率检查

## 5. 事件类型

| 事件类型 | 来源 | 目标 | 功能 |
|---------|------|------|------|
| `STRATEGY_SIGNAL` | StrategyAgent | CoordinatorAgent | 策略信号传递 |
| `RISK_ALERT` | StrategyAgent | CoordinatorAgent | 风险警告 |
| `ORDER_PLACED` | OrderAgent | CoordinatorAgent | 订单已下单 |
| `ORDER_FILLED` | OrderAgent | CoordinatorAgent | 订单已成交 |

## 7. 常见问题排查

### 7.1 策略未生成信号

**检查点**:
1. `_execute_strategy`方法是否有`return signal`
2. `_generate_signal`方法是否返回正确的信号结构

### 7.2 交易未执行

**检查点**:
1. CoordinatorAgent的`_on_strategy_signal`是否接收到信号
2. 预期收益检查是否阻止了交易（检查`skip_return_check`参数）
3. 余额是否充足

### 7.3 事件未传递

**检查点**:
1. EventBus是否正确初始化
2. 智能体是否正确订阅事件
3. 事件回调是否是async函数

## 8. 文件编码规范

- **语言**: 中文注释
- **缩进**: 4空格
- **命名**: 
  - 类名: PascalCase (如`StrategyAgent`)
  - 方法名: snake_case (如`place_order`)
  - 常量: UPPER_SNAKE_CASE (如`MAX_RETRIES`)
- **日志级别**:
  - `DEBUG`: 详细调试信息
  - `INFO`: 一般信息
  - `WARNING`: 警告信息
  - `ERROR`: 错误信息

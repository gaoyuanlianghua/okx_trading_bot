# 策略运行器使用说明

## 概述

策略运行器已将测试脚本的功能整合到正式代码中，提供了统一的策略启动和管理接口。

## 文件结构

```
/root/okx_trading_bot/
├── core/trading/
│   └── strategy_runner.py      # 策略运行器核心模块
├── run_strategy.py              # 正式的策略启动脚本
└── 策略运行器使用说明.md      # 本文档
```

## 核心功能

### 1. 策略运行器 (StrategyRunner)

位置：`core/trading/strategy_runner.py`

主要功能：
- 环境检查和配置加载
- API连接管理和测试
- 策略初始化和启动
- 单次策略执行
- 持续策略运行
- 信号日志记录
- 优雅的停止机制

### 2. 正式启动脚本 (run_strategy.py)

位置：`/root/okx_trading_bot/run_strategy.py`

主要功能：
- 命令行参数解析
- 多策略支持
- 灵活的配置选项
- 单次执行和持续运行模式

## 使用方法

### 基本用法

```bash
# 查看帮助
python3 run_strategy.py --help

# 单次执行核互反动力学策略
python3 run_strategy.py --strategy nuclear_dynamics --once

# 持续运行核互反动力学策略（每60秒）
python3 run_strategy.py --strategy nuclear_dynamics

# 持续运行策略，自定义间隔（每30秒）
python3 run_strategy.py --strategy nuclear_dynamics --interval 30
```

### 可用策略

| 策略名称 | 策略类 | 说明 |
|---------|--------|------|
| nuclear_dynamics | NuclearDynamicsStrategy | 核互反动力学策略 |
| dynamics | DynamicsStrategy | 原子核互反动力学策略 |
| ma_rsi | MARsiStrategy | 移动平均线和RSI策略 |
| macd_bollinger | MacdBollingerStrategy | MACD和布林带策略 |
| arbitrage | ArbitrageStrategy | 套利策略 |
| cross_market_arbitrage | CrossMarketArbitrageStrategy | 跨市场套利策略 |
| machine_learning | MachineLearningStrategy | 机器学习策略 |
| combined | CombinedStrategy | 组合策略 |

### 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| --strategy | 策略名称 | nuclear_dynamics |
| --interval | 执行间隔（秒） | 60 |
| --once | 只执行一次 | False |
| --symbol | 交易对 | BTC-USDT |
| --timeframe | 时间周期 | 1m |

### 示例

#### 1. 单次测试策略

```bash
python3 run_strategy.py --strategy nuclear_dynamics --once
```

#### 2. 持续运行策略

```bash
python3 run_strategy.py --strategy nuclear_dynamics --interval 60
```

#### 3. 使用不同的交易对

```bash
python3 run_strategy.py --strategy ma_rsi --symbol ETH-USDT --once
```

#### 4. 测试多个策略

```bash
# 测试核互反动力学策略
python3 run_strategy.py --strategy nuclear_dynamics --once

# 测试MACD和布林带策略
python3 run_strategy.py --strategy macd_bollinger --once
```

## 策略运行器API

### StrategyRunner 类

```python
from core.trading.strategy_runner import StrategyRunner

runner = StrategyRunner()

# 设置策略
await runner.setup(strategy_class, strategy_config)

# 执行一次策略
signal = await runner.execute_once()

# 持续运行策略
await runner.run_continuous(interval=60)

# 停止策略
await runner.stop()
```

### 主要方法

| 方法 | 说明 |
|------|------|
| setup(strategy_class, strategy_config) | 设置策略运行环境 |
| execute_once() | 执行一次策略 |
| run_continuous(interval) | 持续运行策略 |
| stop() | 停止策略 |

## 日志文件

策略运行器会生成以下日志文件：

- `trading_bot.log` - 主交易日志
- `strategy_runner.log` - 策略运行器日志

## 注意事项

1. **环境切换**：使用 `switch_env.py` 切换实盘/模拟盘环境
2. **API密钥**：确保在对应环境的配置文件中配置正确的API密钥
3. **风险控制**：建议先在模拟盘中充分测试策略后再切换到实盘
4. **监控策略**：定期检查策略执行情况和日志文件

## 进阶使用

### 自定义策略配置

```python
strategy_config = {
    'strategy': {
        'name': 'nuclear_dynamics_strategy',
        'symbol': 'BTC-USDT',
        'timeframe': '1m'
    },
    'params': {
        # 自定义策略参数
        'fall_threshold': 0.02,
        'drift_threshold': 0.001,
        # ... 更多参数
    },
    'risk': {
        'max_leverage': 5,
        'stop_loss': 0.03,
        'take_profit': 0.05
    }
}
```

### 程序化使用策略运行器

```python
import asyncio
from core.trading.strategy_runner import StrategyRunner
from strategies.nuclear_dynamics_strategy import NuclearDynamicsStrategy

async def main():
    runner = StrategyRunner()
    
    try:
        # 设置策略
        strategy_config = {
            'strategy': {
                'name': 'nuclear_dynamics_strategy',
                'symbol': 'BTC-USDT',
                'timeframe': '1m'
            }
        }
        
        await runner.setup(NuclearDynamicsStrategy, strategy_config)
        
        # 持续运行策略
        await runner.run_continuous(interval=60)
        
    except Exception as e:
        print(f"错误: {e}")
    finally:
        await runner.stop()

if __name__ == "__main__":
    loop = asyncio.get_event_loop()
    loop.run_until_complete(main())
    loop.close()
```

## 故障排除

### 问题：Python版本不兼容

解决方案：确保使用 Python 3.6 或更高版本

### 问题：API连接失败

解决方案：
1. 检查网络连接
2. 确认API密钥配置正确
3. 确认在正确的环境（实盘/模拟盘）中

### 问题：策略执行错误

解决方案：
1. 查看日志文件获取详细错误信息
2. 确认策略配置正确
3. 检查市场数据是否正常

## 总结

策略运行器提供了统一、简洁的策略启动和管理接口，将测试脚本的功能整合到正式代码中，使得策略测试和运行更加方便和可靠。

# OKX 交易机器人维护手册

## 1. 项目架构

### 1.1 目录结构

```
okx_trading_bot/
├── main.py                  # 主入口文件
├── trading_gui.py           # GUI 界面
├── okx_api_client.py        # OKX API 客户端
├── okx_websocket_client.py  # OKX WebSocket 客户端
├── agents/                  # 智能体模块
│   ├── base_agent.py
│   ├── decision_coordination_agent.py
│   ├── market_data_agent.py
│   ├── order_agent.py
│   ├── risk_management_agent.py
│   └── strategy_execution_agent.py
├── commons/                 # 公共模块
│   ├── agent_registry.py
│   ├── config_manager.py
│   ├── event_bus.py
│   ├── health_checker.py
│   └── logger_config.py
├── services/                # 服务模块
│   ├── market_data/         # 市场数据服务
│   ├── order_management/    # 订单管理服务
│   └── risk_management/     # 风险管理服务
├── strategies/              # 策略模块
│   ├── base_strategy.py
│   ├── dynamics_strategy.py
│   └── passivbot_integrator.py
├── config/                  # 配置目录
│   └── okx_config.json     # OKX 配置文件
├── requirements.txt         # 依赖列表
├── 使用说明.md              # 使用说明
├── 维护手册.md              # 维护手册
└── README.md                # 项目说明
```

### 1.2 核心模块

- **主入口**：`main.py` - 负责启动应用程序和初始化各个模块
- **GUI 界面**：`trading_gui.py` - 提供用户交互界面
- **API 客户端**：`okx_api_client.py` - 处理 OKX REST API 请求
- **WebSocket 客户端**：`okx_websocket_client.py` - 处理 OKX WebSocket 连接
- **智能体模块**：`agents/` - 实现多智能体架构
- **公共模块**：`commons/` - 提供通用功能和工具
- **服务模块**：`services/` - 实现核心业务逻辑
- **策略模块**：`strategies/` - 实现交易策略

## 2. 系统维护

### 2.1 定期维护任务

#### 2.1.1 日志管理
- **日志位置**：`logs/` 目录
- **日志轮换**：系统会自动按日期和大小轮换日志
- **维护建议**：
  - 定期清理过期日志文件
  - 检查日志文件大小，避免磁盘空间不足
  - 监控错误日志，及时发现和解决问题

#### 2.1.2 配置文件维护
- **配置文件**：`config/okx_config.json`
- **维护建议**：
  - 定期备份配置文件
  - 确保 API 密钥安全
  - 根据市场情况调整策略参数

#### 2.1.3 依赖更新
- **依赖文件**：`requirements.txt`
- **维护建议**：
  - 定期检查依赖库更新
  - 测试更新后的依赖是否兼容
  - 记录依赖版本变更

### 2.2 性能优化

#### 2.2.1 内存管理
- **监控工具**：使用任务管理器或性能监控工具
- **优化建议**：
  - 减少同时监控的交易对数量
  - 调整 WebSocket 连接数量
  - 定期重启应用程序释放内存

#### 2.2.2 网络优化
- **监控指标**：网络延迟、连接稳定性
- **优化建议**：
  - 使用稳定的网络连接
  - 调整 WebSocket 心跳参数
  - 定期检查网络状态

#### 2.2.3 CPU 使用率
- **监控工具**：任务管理器
- **优化建议**：
  - 减少策略计算复杂度
  - 调整数据更新频率
  - 关闭不必要的功能模块
  - 当没有活跃策略时，系统会自动降低线程执行频率

## 3. 故障排查

### 3.1 常见故障

#### 3.1.1 API 连接失败
- **可能原因**：
  - API 密钥错误
  - 网络连接问题
  - OKX API 服务异常
- **排查步骤**：
  1. 检查 API 密钥是否正确
  2. 测试网络连接
  3. 访问 OKX 官网确认服务状态

#### 3.1.2 WebSocket 断开
- **可能原因**：
  - 网络不稳定
  - WebSocket 心跳超时
  - OKX 服务器重启
- **排查步骤**：
  1. 检查网络稳定性
  2. 调整 WebSocket 配置参数
  3. 查看 WebSocket 日志

#### 3.1.3 策略执行异常
- **可能原因**：
  - 策略参数配置错误
  - 市场数据异常
  - 账户余额不足
- **排查步骤**：
  1. 检查策略参数
  2. 查看市场数据
  3. 检查账户余额
  4. 查看策略执行日志

#### 3.1.4 GUI 界面冻结
- **可能原因**：
  - 主线程阻塞
  - 内存不足
  - 大量数据处理
- **排查步骤**：
  1. 等待一段时间，系统可能自动恢复
  2. 检查内存使用情况
  3. 重启应用程序

### 3.2 日志分析

- **日志级别**：DEBUG, INFO, WARNING, ERROR, CRITICAL
- **关键日志**：

## 5. 安全维护

### 5.1 API 密钥安全

- **保护措施**：
  - 不要在代码中硬编码 API 密钥
  - 使用环境变量或配置文件存储 API 密钥
  - 为 API 密钥设置最小权限
  - 定期更换 API 密钥

- **泄露处理**：
  - 立即撤销泄露的 API 密钥
  - 生成新的 API 密钥
  - 更新配置文件
  - 检查账户活动

### 5.2 系统安全

- **保护措施**：
  - 定期更新依赖库

## 11. 策略管理维护

### 11.1 策略添加

- **添加步骤**：
  1. 在 `strategies` 目录下创建新的策略文件
  2. 继承 `BaseStrategy` 类
  3. 实现必要的方法
  4. 点击 "添加策略" 按钮，在弹出的对话框中输入策略名称、策略类名和模块路径
  5. 在策略管理页面选择自定义策略

- **维护建议**：
  - 测试新策略在模拟环境中的表现
  - 监控新策略的执行情况
  - 定期优化策略参数

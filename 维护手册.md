### 1.2 核心模块

- **主入口**：`main.py` - 负责启动应用程序和初始化各个模块
- **GUI 界面**：`trading_gui.py` - 提供用户交互界面，已优化样式和功能
- **简化版 GUI 界面**：`trading_gui_simple.py` - 提供简化版用户界面
- **核心智能体**：`core/agents/` - 实现多智能体架构，包括协调器、市场数据、订单、风险管理和账户同步智能体
- **API 模块**：`core/api/` - 实现 API 管理、响应解析和 OKX REST 客户端
- **工具模块**：`core/utils/` - 提供循环事件管理器和盈利增长管理器
- **策略模块**：`strategies/` - 实现交易策略，包括核互反动力学策略
- **脚本模块**：`scripts/` - 提供生成交易日志和定时 API 调用功能
- **无头模式**：`start_headless.py` - 支持无界面运行模式
- **测试工具**：`test_place_order.py` 和 `check_orders.py` - 用于测试订单功能
- **日志模块**：`logs/` - 存储系统日志

## 3. 故障排查

### 3.1 常见故障

#### 3.1.1 API 连接失败
- **可能原因**：
  - API 密钥错误
  - 网络连接问题
  - OKX API 服务异常
- **排查步骤**：
  1. 检查 API 密钥是否正确
  2. 测试网络连接
  3. 访问 OKX 官网确认服务状态

#### 3.1.2 WebSocket 断开
- **可能原因**：
  - 网络不稳定
  - WebSocket 心跳超时
  - OKX 服务器重启
- **排查步骤**：
  1. 检查网络稳定性
  2. 调整 WebSocket 配置参数
  3. 查看 WebSocket 日志

#### 3.1.3 策略执行异常
- **可能原因**：
  - 策略参数配置错误
  - 市场数据异常
  - 账户余额不足
- **排查步骤**：
  1. 检查策略参数
  2. 查看市场数据
  3. 检查账户余额
  4. 查看策略执行日志

#### 3.1.4 GUI 界面冻结
- **可能原因**：
  - 主线程阻塞
  - 内存不足
  - 大量数据处理
- **排查步骤**：
  1. 等待一段时间，系统可能自动恢复
  2. 检查内存使用情况
  3. 重启应用程序

## 4. 代码维护

### 4.1 代码结构

#### 4.1.1 智能体架构
- **BaseAgent**：所有智能体的基类
- **MarketDataAgent**：处理市场数据
- **OrderAgent**：处理订单操作，包括订单同步
- **RiskManagementAgent**：处理风险管理
- **CoordinatorAgent**：协调各个智能体，实现交易决策
- **AccountSyncAgent**：同步账户信息，包括余额、持仓和订单
- **AgentManager**：管理智能体注册和消息分发

#### 4.1.2 API 模块
- **APIManager**：管理 API 调用和响应解析，实现消息分发
- **APIResponseParser**：解析 OKX API 响应数据
- **OKXRESTClient**：处理 OKX REST API 请求，实现订单放置和取消

#### 4.1.3 工具模块
- **CycleEventManager**：循环事件管理器，负责主循环事件，利用网络延迟监控实现 API 时间和本地时间的双轨道运行
- **ProfitGrowthManager**：盈利增长管理器，确保每笔交易实现 BTC 和 USDT 相互增长

#### 4.1.4 策略层
- **BaseStrategy**：所有策略的基类
- **NuclearDynamicsStrategy**：核互反动力学策略，基于物理模型生成交易信号，实现数值信号评分
- **DynamicsStrategy**：动态调整策略
- **PassivbotIntegrator**：集成 Passivbot 策略

#### 4.1.5 盈利增长管理层
- **ProfitGrowthManager** (`core/utils/profit_growth_manager.py`)
  - **职责**：验证每笔交易是否满足盈利条件，确保 BTC 和 USDT 相互增长
  - **核心功能**：
    - 买入条件验证：价格必须低于上次卖出价 0.7% 以上
    - 卖出条件验证：收益率必须超过 0.7%
    - 平均买入价计算和跟踪
    - 累计盈利统计
    - 支持杠杆交易的手续费计算
    - 支持空头持仓的计算
  - **状态文件**：`data/profit_growth_state.json`
  - **关键参数**：
    - `min_profit_rate = 0.005` (0.5% 最小盈利率)
    - `fee_rate = 0.001` (0.1% 单边手续费率)
  - **协作关系**：
    - 接收来自 NuclearDynamicsStrategy 的交易信号
    - 向 CoordinatorAgent 返回验证结果
    - 与 OrderAgent 协作记录交易历史
    - 与 AccountSyncAgent 同步账户信息

#### 4.1.6 脚本模块
- **generate_trade_logs.py**：生成交易日志，用于测试和调试
- **schedule_api_logs.py**：定时 API 调用，实现测试订单放置和取消

## 5. 安全维护

### 5.1 API 密钥安全

- **保护措施**：
  - 不要在代码中硬编码 API 密钥

# OKX 交易机器人使用说明

## 1. 项目概述

OKX 交易机器人是一个基于事件驱动的智能体系统，支持 OKX 交易所的 REST API 和 WebSocket 连接。该项目采用新的架构设计，包含市场数据、订单管理、风险管理和策略执行等核心模块，专注于交易和收益跟踪功能。

### 1.1 最新版本特性

- **事件驱动架构**：基于事件总线的智能体通信系统
- **智能体系统**：包含市场数据、订单管理、风险管理、策略执行、协调智能体和账户同步智能体
- **盈利增长管理器**：确保每笔交易都实现BTC和USDT相互增长
  - 买入条件验证：价格必须低于上次卖出价0.7%以上
  - 卖出条件验证：收益率必须超过0.7%（覆盖手续费+盈利）
  - 状态持久化：自动保存交易状态到本地和OSS
- **交易跟踪**：支持交易历史和收益跟踪
- **账户管理**：支持账户信息和资产分布管理
- **风险管理**：实现风险管理和风险警报功能
- **策略集成**：集成原子核互反动力学策略
- **交易摘要**：实现交易摘要功能
- **测试验证**：提供完整的测试脚本
- **实时监控**：策略运行监控、API 调用监控和日志系统
- **WebSocket GUI**：基于 PyQt5 的实时交易界面
- **详细的 API 监控**：API 调用和响应的详细记录
- **策略管理**：支持创建、启动、停止和管理策略
- **OSS数据持久化**：自动备份交易数据到阿里云OSS
- **循环事件管理**：负责主循环事件，利用监控网络延时实现 API 时间和本地时间的双轨道运行，实现定时任务和数据校准
- **消息分发功能**：API 管理器实现了消息分发功能，支持按智能体类型分发消息
- **智能体消息处理**：各智能体在接收到来自 API 管理器的信息和进行任务时，能够快速逐条处理接收到的消息
- **协调器通信**：当需要发出请求时，由协调器智能体进行通信，将向外的信息传递到向外的工具
- **秒级信号处理**：系统支持 POST、WebSocket、REST、HTTP 进行秒级信号处理和分发
- **账户同步智能体**：负责同步账户信息，包括余额、持仓、订单等，实现与交易所的实时数据同步
- **订单同步功能**：订单智能体实现了订单同步功能，从交易所同步订单信息，确保系统数据与交易所数据保持一致
- **测试挂单功能**：实现了每 10 分钟放置一个测试挂单，1 分钟后自动撤销的功能，确保账户活跃度

## 3. 安装与配置

### 3.1 安装步骤
1. 克隆或下载项目代码到本地
2. 打开命令行终端，进入项目目录
3. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
4. 配置 API 密钥：
   - **GUI模式**：在启动界面后直接输入 API 密钥信息
   - **命令行模式**：通过环境变量设置 API 密钥

### 3.2 配置说明

#### 3.2.1 API 配置

**GUI模式**：
1. 启动 `websocket_gui.py`
2. 在顶部输入框中输入 API Key、API Secret 和 Passphrase
3. 选择模拟盘或实盘环境
4. 点击 "连接" 按钮

**命令行模式**：
在 `main_new.py` 文件中修改以下配置：

```python
# 获取API配置
api_key = os.getenv('OKX_API_KEY', '')
api_secret = os.getenv('OKX_API_SECRET', '')
passphrase = os.getenv('OKX_PASSPHRASE', '')
is_test = True  # True 为模拟盘，False 为实盘
```

#### 3.2.2 策略配置

在 `strategies/dynamics_strategy.py` 文件中修改策略参数：

```python
# 动力学参数配置
self.dynamics_params = {
    'ε': 0.85,       # 市场动量方向算符 (1同向/-1反向)
    'G_eff': 1.2e-3, # 市场耦合系数 (10⁻³~10⁻²量级)
    'n': 3,          # 市场影响力衰减指数
    'η': 0.75,       # 配对反馈强度
    'γ': 0.1,        # 价差异常衰减率
    'κ': 2.5,        # 相位同步强度
    'λ': 3.0,        # 相位耦合衰减
    't_coll': 0.1    # 市场协同时标(秒)
}
```

## 4. 运行与使用

### 4.1 启动机器人

#### 4.1.1 GUI模式
1. 运行 GUI 程序：
   ```bash
   python websocket_gui.py
   ```
2. 在顶部输入框中输入您的 OKX API 密钥、密钥密码和密码短语
3. 选择模拟盘或实盘环境
4. 点击 "连接" 按钮连接到 OKX 服务器
5. 连接成功后，系统会自动订阅默认产品（BTC-USDT-SWAP、ETH-USDT-SWAP）

#### 4.1.2 命令行模式
1. 运行命令行程序：
   ```bash
   python main_new.py
   ```
2. 系统会自动初始化所有组件并启动交易机器人

## 6. 常见问题与解决方案

### 6.1 连接问题

**问题**：API 连接失败
**解决方案**：
- 检查 API 密钥是否正确
- 检查网络连接
- 检查 OKX API 服务状态

**问题**：WebSocket 断开连接
**解决方案**：
- 检查网络稳定性
- 系统会自动重连

**问题**：点击连接按钮界面自动退出
**解决方案**：
- 确保输入了正确的 API 密钥信息
- 检查网络连接是否稳定
- 系统已修复此问题，现在会在后台处理异步任务

### 6.2 交易问题

**问题**：订单无法执行
**解决方案**：
- 检查账户余额
- 检查交易对是否支持
- 检查订单参数是否合法

**问题**：策略执行异常
**解决方案**：
- 检查策略参数配置
- 查看日志文件
- 重启策略

### 6.3 系统问题

**问题**：GUI 冻结
**解决方案**：
- 等待一段时间，系统会自动恢复
- 或重启程序

**问题**：内存占用过高
**解决方案**：
- 关闭不必要的功能
- 减少监控的交易对数量
- 定期重启程序

### 6.4 策略问题

**问题**：如何添加自定义策略？
**解决方案**：
- 在策略管理页面点击 "创建策略" 按钮
- 输入策略名称和类型
- 在弹出的对话框中粘贴策略代码
- 点击 "创建" 按钮

**问题**：策略目录未找到
**解决方案**：
- 系统已修复此问题，现在会自动创建策略目录
- 策略目录位于 `d:\Projects\okx_trading_bot\strategies`

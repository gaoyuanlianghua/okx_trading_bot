# OKX 交易机器人使用说明

## 1. 项目概述

OKX 交易机器人是一个基于 PyQt5 开发的 GUI 交易机器人，支持 OKX 交易所的 REST API 和 WebSocket 连接，并提供 Socks5 代理支持。该项目采用多智能体架构设计，包含市场数据、订单管理、风险管理和策略执行等核心模块。

## 2. 系统要求

### 2.1 硬件要求
- 处理器：至少 Intel Core i3 或同等性能
- 内存：至少 2GB RAM
- 磁盘空间：至少 100MB 可用空间
- 网络：稳定的互联网连接

### 2.2 软件要求
- 操作系统：Windows 10/11
- Python 版本：3.8 或更高版本
- 依赖库：详见 `requirements.txt`

## 3. 安装与配置

### 3.1 安装步骤
1. 克隆或下载项目代码到本地
2. 打开命令行终端，进入项目目录
3. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
4. 配置 API 密钥：
   - 复制 `config.ini.example` 为 `config.ini`
   - 编辑 `config.ini` 文件，填写 OKX API 密钥信息

### 3.2 配置文件说明

#### 3.2.1 API 配置

```ini
[api]
api_key = your_api_key
secret_key = your_secret_key
passphrase = your_passphrase
proxy_type = socks5
proxy_host = 127.0.0.1
proxy_port = 1080
proxy_username = 
proxy_password = 
```

#### 3.2.2 WebSocket 配置

```ini
[websocket]
enable = True
ping_interval = 30
ping_timeout = 10
max_reconnect_attempts = 5
reconnect_delay = 2
```

#### 3.2.3 GUI 配置

```ini
[gui]
enable_sound = True
auto_start = False
```

## 4. 运行与使用

### 4.1 启动机器人
1. 运行主程序：
   ```bash
   python main.py
   ```
2. 启动后会出现登录界面，输入您的 OKX API 密钥信息
3. 点击 "登录" 按钮进入主界面

### 4.2 主界面功能

#### 4.2.1 市场数据页面
- 显示实时行情数据
- 支持多币种监控
- 提供K线图表

#### 4.2.2 交易页面
- 支持市价单和限价单
- 显示当前持仓
- 提供交易历史记录

#### 4.2.3 策略页面
- 支持 Passivbot 策略配置
- 提供策略参数调整
- 显示策略运行状态

#### 4.2.4 日志页面
- 显示系统运行日志
- 支持日志级别过滤
- 提供日志导出功能

#### 4.2.5 健康检查页面
- 显示系统和网络状态
- 提供 API 连接测试
- 显示 WebSocket 连接状态

### 4.3 交易操作

#### 4.3.1 手动交易
1. 在交易页面选择交易对
2. 输入交易数量
3. 选择交易类型（买入/卖出）
4. 选择订单类型（市价/限价）
5. 点击 "执行" 按钮

#### 4.3.2 策略交易
1. 在策略页面选择策略类型
2. 配置策略参数
3. 点击 "启动策略" 按钮
4. 监控策略运行状态

### 4.4 监控与管理

- **实时监控**：通过主界面实时查看市场数据和交易状态
- **日志查看**：通过日志页面查看系统运行情况
- **健康检查**：定期检查系统和网络状态
- **手动干预**：在必要时手动干预交易

## 5. 策略使用

### 5.1 Passivbot 策略

Passivbot 是一种基于网格交易的策略，具有以下特点：
- 自动设置网格订单
- 动态调整仓位
- 支持多种市场条件

#### 5.1.1 配置 Passivbot
1. 在策略页面选择 "Passivbot"
2. 配置以下参数：
   - 交易对
   - 初始资金
   - 网格参数
   - 风险参数
3. 点击 "保存配置" 按钮

#### 5.1.2 运行 Passivbot
1. 点击 "启动策略" 按钮
2. 监控策略运行状态
3. 可随时调整策略参数

### 5.2 自定义策略

1. 在 `strategies` 目录下创建新的策略文件
2. 继承 `BaseStrategy` 类
3. 实现必要的方法
4. 在策略页面选择自定义策略

## 6. 常见问题与解决方案

### 6.1 连接问题

**问题**：API 连接失败
**解决方案**：
- 检查 API 密钥是否正确
- 检查网络连接
- 检查代理配置
- 检查 OKX API 服务状态

**问题**：WebSocket 断开连接
**解决方案**：
- 检查网络稳定性
- 调整 WebSocket 配置参数
- 系统会自动重连

### 6.2 交易问题

**问题**：订单无法执行
**解决方案**：
- 检查账户余额
- 检查交易对是否支持
- 检查订单参数是否合法

**问题**：策略执行异常
**解决方案**：
- 检查策略参数配置
- 查看日志文件
- 重启策略

### 6.3 系统问题

**问题**：GUI 冻结
**解决方案**：
- 等待一段时间，系统会自动恢复
- 或重启程序

**问题**：内存占用过高
**解决方案**：
- 关闭不必要的功能
- 减少监控的交易对数量
- 定期重启程序

- **监控异常**：定期检查交易记录，发现异常及时处理

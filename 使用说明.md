# OKX 交易机器人使用说明

## 1. 项目概述

OKX 交易机器人是一个基于事件驱动的智能体系统，支持 OKX 交易所的 REST API 和 WebSocket 连接。该项目采用新的架构设计，包含市场数据、订单管理、风险管理和策略执行等核心模块，专注于交易和收益跟踪功能。

### 1.1 最新版本特性

- **事件驱动架构**：基于事件总线的智能体通信系统
- **智能体系统**：包含市场数据、订单管理、风险管理、策略执行和协调智能体
- **交易跟踪**：支持交易历史和收益跟踪
- **账户管理**：支持账户信息和资产分布管理
- **风险管理**：实现风险管理和风险警报功能
- **策略集成**：集成原子核互反动力学策略
- **交易摘要**：实现交易摘要功能
- **测试验证**：提供完整的测试脚本

## 1.2 版本历史

- **v2.0.0**：架构重构、核心模块实现、智能体系统、事件系统、交易功能、账户管理、风险管理、策略集成、交易摘要
- **v1.4.0**：策略交易信息、订单信息、手动规划、窗口管理、布局优化、内容缩放、网络状态、后台运行

## 2. 系统要求

### 2.1 硬件要求
- 处理器：至少 Intel Core i3 或同等性能
- 内存：至少 2GB RAM
- 磁盘空间：至少 100MB 可用空间
- 网络：稳定的互联网连接

### 2.2 软件要求
- 操作系统：Windows 10/11
- Python 版本：3.8 或更高版本
- 依赖库：详见 `requirements.txt`
  - 新增依赖：PyQt5, cryptography, hjson

## 3. 安装与配置

### 3.1 安装步骤
1. 克隆或下载项目代码到本地
2. 打开命令行终端，进入项目目录
3. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
4. 配置 API 密钥：
   - 在 `main_new.py` 文件中修改 API 配置
   - 或通过环境变量设置 API 密钥

### 3.2 配置说明

#### 3.2.1 API 配置

在 `main_new.py` 文件中修改以下配置：

```python
# 获取API配置
api_config = config.get('api', {})
api_key = api_config.get('api_key', os.getenv('OKX_API_KEY', ''))
api_secret = api_config.get('api_secret', os.getenv('OKX_API_SECRET', ''))
passphrase = api_config.get('passphrase', os.getenv('OKX_PASSPHRASE', ''))
is_test = api_config.get('is_test', True)
```

#### 3.2.2 策略配置

在 `strategies/dynamics_strategy.py` 文件中修改策略参数：

```python
# 动力学参数配置
self.dynamics_params = {
    'ε': 0.85,       # 市场动量方向算符 (1同向/-1反向)
    'G_eff': 1.2e-3, # 市场耦合系数 (10⁻³~10⁻²量级)
    'n': 3,          # 市场影响力衰减指数
    'η': 0.75,       # 配对反馈强度
    'γ': 0.1,        # 价差异常衰减率
    'κ': 2.5,        # 相位同步强度
    'λ': 3.0,        # 相位耦合衰减
    't_coll': 0.1    # 市场协同时标(秒)
}
```

## 4. 运行与使用

### 4.1 启动机器人
1. 运行主程序：
   ```bash
   python main.py
   ```
2. 系统会自动加载功能，无需手动点击"加载功能"按钮
3. 进入主界面后，在"配置管理"标签页中输入您的 OKX API 密钥信息
4. 点击 "保存配置" 按钮

### 4.2 主界面功能

#### 4.2.1 市场数据页面
- 显示实时行情数据
- 支持多币种监控
- 提供K线图表

#### 4.2.2 交易页面
- 支持市价单和限价单
- 显示当前持仓
- 提供交易历史记录

#### 4.2.3 策略管理页面
- 支持策略配置和管理
- 提供策略参数调整
- 显示策略运行状态
- 支持添加自定义策略

#### 4.2.4 日志页面
- 显示系统运行日志
- 支持日志级别过滤
- 提供日志导出功能

#### 4.2.5 网络状态页面
- 显示系统和网络状态
- 提供 API 连接测试
- 显示 WebSocket 连接状态
- 支持测试网和主网切换

### 4.3 交易操作

#### 4.3.1 手动交易
1. 在交易页面选择交易对
2. 输入交易数量
3. 选择交易类型（买入/卖出）
4. 选择订单类型（市价/限价）
5. 点击 "执行" 按钮

#### 4.3.2 策略交易
1. 在策略管理页面选择策略类型
2. 配置策略参数
3. 点击 "激活策略" 按钮
4. 监控策略运行状态

### 4.4 监控与管理

- **实时监控**：通过主界面实时查看市场数据和交易状态
- **日志查看**：通过日志页面查看系统运行情况
- **健康检查**：定期检查系统和网络状态
- **手动干预**：在必要时手动干预交易

## 5. 策略使用

### 5.1 原子核互反动力学策略

原子核互反动力学策略是一种基于市场波动的动态网格策略，具有以下特点：
- 基于市场波动的动态网格策略
- 支持多时间周期
- 自动调整网格间距和倍数

#### 5.1.1 配置原子核互反动力学策略
1. 在策略管理页面选择 "DynamicsStrategy"
2. 配置策略参数
3. 点击 "保存参数" 按钮

#### 5.1.2 运行原子核互反动力学策略
1. 点击 "激活策略" 按钮
2. 监控策略运行状态
3. 可随时调整策略参数

### 5.2 自定义策略

1. 在 `strategies` 目录下创建新的策略文件
2. 继承 `BaseStrategy` 类
3. 实现必要的方法
4. 点击 "添加策略" 按钮，在弹出的对话框中输入策略名称、策略类名和模块路径
5. 在策略管理页面选择自定义策略

## 6. 常见问题与解决方案

### 6.1 连接问题

**问题**：API 连接失败
**解决方案**：
- 检查 API 密钥是否正确
- 检查网络连接
- 检查 OKX API 服务状态

**问题**：WebSocket 断开连接
**解决方案**：
- 检查网络稳定性
- 调整 WebSocket 配置参数
- 系统会自动重连

### 6.2 交易问题

**问题**：订单无法执行
**解决方案**：
- 检查账户余额
- 检查交易对是否支持
- 检查订单参数是否合法

**问题**：策略执行异常
**解决方案**：
- 检查策略参数配置
- 查看日志文件
- 重启策略

### 6.3 系统问题

**问题**：GUI 冻结
**解决方案**：
- 等待一段时间，系统会自动恢复
- 或重启程序

**问题**：内存占用过高
**解决方案**：
- 关闭不必要的功能
- 减少监控的交易对数量
- 定期重启程序

### 6.4 策略问题

**问题**：如何添加自定义策略？
**解决方案**：
- 点击 "添加策略" 按钮
- 在弹出的对话框中输入策略名称、策略类名和模块路径
- 点击 "添加" 按钮

**问题**：为什么没有"加载功能"按钮？
**解决方案**：
- 系统现在会在启动时自动加载功能，无需手动点击按钮

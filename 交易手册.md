在 `config/config.yaml` 文件中配置API密钥：

```yaml
api:

  api_secret: your_api_secret
  passphrase: your_passphrase
  is_test: true  # 模拟盘为true，实盘为false
```

## 常见问题和解决方案

### API下单失败

#### 问题1：合约交易失败

- **错误代码**：51121
- **错误信息**："Order quantity must be a multiple of the lot size."
- **原因**：合约交易的最小交易单位是1，不是0.001
- **解决方案**：使用现货交易

#### 问题2：账户模式不支持

- **错误代码**：51010
- **错误信息**："You can't complete this request under your current account mode."
- **原因**：账户未开通合约交易权限
- **解决方案**：使用现货交易

#### 问题3：API连接失败

- **原因**：网络问题或API密钥错误
- **解决方案**：检查网络连接和API配置

### 交易执行问题

#### 信号未转化为交易

- **原因**：信号强度未达到阈值
- **解决方案**：调整策略阈值参数

#### 交易执行失败

- **原因**：账户余额不足或API权限问题
- **解决方案**：检查账户余额和API权限

## 商品交易要求

### 基本交易规则

**未成交订单限制**:

- 未成交订单（包括 post only，limit和处理中的taker单）的最大挂单数：4,000个
- 单个交易产品未成交订单的最大挂单数：500个

**被计入挂单数量限制的订单类型**:

- 限价委托 (Limit)
- 市价委托 (Market)
- 只挂单 (Post only)
- 全部成交或立即取消 (FOK)
- 立即成交并取消剩余 (IOC)
- 市价委托立即成交并取消剩余 (optimal limit IOC)
- 止盈止损 (TP/SL)
- 以下类型的订单触发的限价和市价委托：
  - 止盈止损 (TP/SL)
  - 计划委托 (Trigger)
  - 移动止盈止损 (Trailing stop)
  - 套利下单 (Arbitrage)
  - 冰山策略 (Iceberg)
  - 时间加权策略 (TWAP)
  - 定投 (Recurring buy)

**策略委托订单最大挂单数**:

- 止盈止损：100个 每个Instrument ID
- 计划委托：500个
- 移动止盈止损：50个
- 冰山委托：100个
- 时间加权委托：20个

**网格策略最大个数**:

- 现货网格：100个
- 合约网格：100个

**交易限制规则**:

- 当taker订单匹配的maker订单数量超过最大限制1000笔时，taker订单将被取消
- 限价单仅成交与1000笔maker订单相对应的部分，并取消剩余
- 全部成交或立即取消（FOK）订单将直接被取消

### 交易产品基础信息

**产品类型**:

- **SPOT**：币币
- **MARGIN**：币币杠杆
- **SWAP**：永续合约
- **FUTURES**：交割合约
- **OPTION**：期权

**交易产品参数**:

- **baseCcy**：交易货币币种，如 BTC-USDT 中的 BTC（仅适用于币币/币币杠杆）
- **quoteCcy**：计价货币币种，如 BTC-USDT 中的 USDT（仅适用于币币/币币杠杆）
- **settleCcy**：盈亏结算和保证金币种（仅适用于交割/永续/期权）
- **ctVal**：合约面值（仅适用于交割/永续/期权）
- **ctMult**：合约乘数（仅适用于交割/永续/期权）
- **tickSz**：下单价格精度，如 0.0001
- **lotSz**：下单数量精度
- **minSz**：最小下单数量
- **ctType**：合约类型（linear：正向合约，inverse：反向合约）
- **state**：产品状态（live：交易中，suspend：暂停中，preopen：预上线，test：测试中）

**单笔委托限制**:

- **maxLmtSz**：限价单的单笔最大委托数量
- **maxMktSz**：市价单的单笔最大委托数量
- **maxLmtAmt**：限价单的单笔最大美元价值
- **maxMktAmt**：市价单的单笔最大美元价值
- **maxTwapSz**：时间加权单的单笔最大委托数量（单笔最小委托数量为 minSz\*2）
- **maxIcebergSz**：冰山委托的单笔最大委托数量
- **maxTriggerSz**：计划委托委托的单笔最大委托数量
- **maxStopSz**：止盈止损市价委托的单笔最大委托数量

## 合约交易最小值

### 重要注意事项

**合约交易最小单位**:

- **BTC-USDT-SWAP**: 1个合约
- **ETH-USDT-SWAP**: 1个合约
- **LTC-USDT-SWAP**: 1个合约
- **BNB-USDT-SWAP**: 1个合约
- **SOL-USDT-SWAP**: 1个合约
- **ADA-USDT-SWAP**: 1个合约
- **错误代码 51121**: "Order quantity must be a multiple of the lot size." 表示订单数量必须是最小单位的倍数
- **建议**: 所有合约交易使用1个合约作为最小交易量

**现货交易最小单位**:

- **BTC-USDT**: 0.001 BTC
- **ETH-USDT**: 0.001 ETH
- **LTC-USDT**: 0.01 LTC
- **BNB-USDT**: 0.01 BNB
- **SOL-USDT**: 0.01 SOL
- **ADA-USDT**: 1 ADA
- **建议**: 现货交易使用对应品种的最小交易单位

### 杠杆交易配置

**杠杆倍数**:

- 支持1-3倍杠杆
- 建议使用2倍杠杆作为平衡点

**交易模式**:

- **逐仓 (isolated)**: 风险可控，适合初学者
- **全仓 (cross)**: 风险较大，但资金利用率高

**资金管理**:

- 总交易金额建议控制在100USDT以内
- 单次交易金额根据杠杆倍数调整
- **协调器智能体交易金额计算逻辑**:
  - 对于BTC-USDT-SWAP合约，1个合约的价值约为1美元
  - 设定单次交易金额为0.5 USDT，适应当前账户资金水平
  - 每次交易后，总交易金额会增加0.5 USDT
  - 当总交易金额达到或超过100 USDT时，交易会被暂停

## 盈利增长管理器

### 功能概述

盈利增长管理器是确保每笔交易都实现BTC和USDT相互增长的核心组件。它与核互反动力学策略协作，验证交易信号是否满足盈利条件。

### 核心机制

#### 买入条件

- **首次买入**：无历史卖出记录时允许买入
- **后续买入**：当前价格必须低于上次卖出价格至少 **0.7%**
  - 0.2%：覆盖双边手续费（买入0.1% + 卖出0.1%）
  - 0.5%：最小盈利要求

#### 卖出条件

- 收益率必须超过 **0.7%** 才能执行卖出
- 计算公式：`(当前价格 - 平均买入价) / 平均买入价 > 0.7%`

#### 状态持久化

- 自动保存到 `data/profit_growth_state.json`
- 包含：累计盈利、交易次数、上次卖出价格、平均买入价格、持有BTC数量
- 重启后自动加载历史状态

### 查看盈利状态

```bash

# 查看交易日志

```

## 交易策略配置

### 策略选择

在 `config/config.yaml` 中设置默认策略：

```yaml
strategy:
  default_strategy: NuclearDynamicsStrategy
  strategies:
    NuclearDynamicsStrategy:
      G_eff: 0.0015        # 市场耦合系数
      max_position: 0.5    # 最大仓位比例
      position_size: 0.1   # 仓位大小
      stop_loss: 0.02      # 止损比例
      take_profit: 0.05    # 止盈比例
      ε: 0.85              # 市场动量方向算符
```

### 策略与盈利增长管理器的协作流程

```
┌─────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│  核互反动力学策略  │────►│   盈利增长管理器   │────►│    协调代理       │
│                 │     │                 │     │                 │
│ 生成买卖信号     │     │ 验证盈利条件      │     │ 执行/拒绝交易     │
└─────────────────┘     └──────────────────┘     └──────────────────┘
        │                        │                        │
        │                        │                        │
        ▼                        ▼                        ▼
  基于物理模型分析          检查收益率阈值            更新账户状态
  (弹簧飘移/角动量流)       (买入: >0.7%跌幅)         记录交易历史
                           (卖出: >0.7%收益)
```

### 策略参数调整

- **信号阈值**：默认为±0.5，可根据市场情况调整
- **交易频率**：根据策略执行时间间隔调整
- **风险参数**：设置合理的止损和止盈比例
- **盈利门槛**：在 `profit_growth_manager.py` 中调整 `min_profit_rate` 和 `fee_rate`

### 策略监控

- 定期检查策略表现
- 根据市场变化调整参数
- 监控策略信号强度分布
- 查看盈利增长管理器的累计盈利统计

## 订单执行与智能体参数传递

### 智能体架构

| 智能体名称                     | 职责                             | 主要参数                                                           | 传递方式   |
| ------------------------- | ------------------------------ | -------------------------------------------------------------- | ------ |
| 协调智能体 (CoordinatorAgent)  | 管理所有智能体的生命周期，协调智能体间的通信，处理系统级决策 | max\_trade\_amount, auto\_recovery, emergency\_stop\_threshold | 内部配置   |
| 市场数据智能体 (MarketDataAgent) | 收集和处理市场数据，提供实时价格和K线数据          | exchange\_name, api\_key, api\_secret, passphrase, is\_test    | 构造函数参数 |

| 风险智能体 (RiskAgent)         | 监控账户风险，提供风险评估和预警               | rest\_client                                                   | 构造函数参数 |
| 策略智能体 (StrategyAgent)     | 管理和执行交易策略，生成交易信号               | market\_data\_agent, order\_agent                              | 构造函数参数 |

### 订单执行流程

1. **策略信号生成**：策略分析市场数据生成交易信号
2. **信号传递**：信号通过事件总线传递给协调器
3. **交易执行**：协调器调用订单智能体执行交易
4. **订单管理**：监控订单状态和执行结果
5. **风险控制**：实时监控账户和交易风险

| lever     | 杠杆交易必须提供杠杆倍数           | 1     | 检查杠杆参数    |

## 测试挂单功能

### 功能概述

系统实现了每 10 分钟放置一个测试挂单，1 分钟后自动撤销的功能，确保账户活跃度。

### 核心功能

- **测试挂单**：每 10 分钟放置一个测试挂单
- **自动撤销**：1 分钟后自动撤销测试挂单
- **账户活跃度**：确保账户活跃度
- **错误处理**：处理挂单和撤单过程中的错误

### 使用方法

```python
# 放置测试挂单
async def place_test_order(self):
    try:
        # 获取当前行情数据
        ticker_info = await self.api_manager.get_ticker("BTC-USDT")
        if not ticker_info:
            print("获取行情数据失败，无法放置测试挂单")
            return
        
        # 获取当前价格
        last_price = ticker_info.get("last_price")
        if not last_price:
            print("获取价格失败，无法放置测试挂单")
            return
        
        # 计算挂单价格（上下浮动 0.3%）
        buy_price = last_price * 0.997
        sell_price = last_price * 1.003
        
        # 放置买单
        buy_order_id = await self.api_manager.place_order(
            inst_id="BTC-USDT",
            side="buy",
            ord_type="limit",
            sz="0.00001",
            px=str(buy_price),
            td_mode="cash"
        )
        
        # 放置卖单
        sell_order_id = await self.api_manager.place_order(
            inst_id="BTC-USDT",
            side="sell",
            ord_type="limit",
            sz="0.00001",
            px=str(sell_price),
            td_mode="cash"
        )
        
        print(f"放置测试挂单成功: 买单 ID: {buy_order_id}, 卖单 ID: {sell_order_id}")
        
        # 1分钟后自动撤单
        await asyncio.sleep(60)
        
        # 撤销买单
        if buy_order_id:
            await self.api_manager.cancel_order("BTC-USDT", buy_order_id)
            print(f"已撤销买单: {buy_order_id}")
        
        # 撤销卖单
        if sell_order_id:
            await self.api_manager.cancel_order("BTC-USDT", sell_order_id)
            print(f"已撤销卖单: {sell_order_id}")
            
    except Exception as e:
        print(f"放置测试挂单失败: {e}")
```
